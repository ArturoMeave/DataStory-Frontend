## Environment

The app requires `VITE_API_URL` pointing to the backend (defaults to `http://localhost:3001`). The backend lives in a separate repository, **DataStory-Backend**, built with Node.js, Express, Prisma, and PostgreSQL. It must be running for AI features (summary, tasks, chat) and auth to work.

## Architecture

**DataStory** is a financial data dashboard for small businesses. Users upload a CSV (or connect Stripe), then get AI-driven analysis, anomaly detection, and forecasting.

### Routing (`src/App.tsx`)
- `/` — public landing page
- `/auth` — login/register
- `/dashboard` — main app (protected)
- `/share/:id` — read-only shared snapshot (protected)

`ProtectedRoute` checks `useAuthStore` and redirects to `/auth` if unauthenticated.

### State Management (`src/stores/`)
Two Zustand stores:
- **`authStore`** — JWT token + user, persisted to `localStorage` under `datastory_token` / `datastory_user`. Auth is read directly from `localStorage` in `api.service.ts` to avoid circular imports.
- **`dataStore`** — all dashboard state: parsed CSV rows, goal, tasks, chat history, AI summary, anomalies, and read-only flag.

### Dashboard Layout (`src/pages/DashboardPage.tsx`)
`DashboardPage` is the main component of the application. It renders a `Navbar` at the top and, when data is loaded, a two-column grid layout: a main content area (`1fr`) on the left and a fixed-width sidebar (`340px`) on the right. The sidebar holds `TaskPanel` and `ChatPanel`. Visual/widget components live in `src/components/dashboard/` and chart components in `src/components/charts/`.

### Data Flow
1. User drops a CSV or connects Stripe → data lands in `dataStore.rows`
2. `DashboardPage` `useEffect` on `rows`/`goal` → runs `detectAnomalies()` and calls `/api/ai/generate` for an AI summary
3. Charts (`RevenueLineChart`, `ExpenseBarChart`) read from `dataStore` and append 3 forecast points generated by `generateForecast()` (linear regression in `forecastEngine.ts`)

### Services (`src/services/`)
- **`api.service.ts`** — wraps all backend calls. Single `request()` helper injects Bearer token. AI endpoints (`generateSummary`, `generateTasks`, `chatWithData`) all hit `POST /api/ai/generate` with different system prompts in Spanish. `saveSnapshot` saves dashboard state to `POST /api/snapshots`.
- **`stripe.service.ts`** — calls Stripe API directly from the browser with the user's secret key to fetch the last 90 days of charges.
- **`pdf.service.ts`** — generates a PDF report using jsPDF + html2canvas, capturing charts by DOM id (`chart-revenue`, `chart-expenses`).

### Utils (`src/utils/`)
- **`csvParser.ts`** — PapaParse wrapper with fuzzy column detection (supports Spanish/English column names)
- **`dataAggregator.ts`** — pure aggregation functions + `buildDataSummary()` which builds the text sent to the AI (compressed summary, not raw CSV)
- **`anomalyDetector.ts`** — z-score based (threshold: 2.0 std devs)
- **`forecastEngine.ts`** — simple linear regression projecting 3 future periods

### Styling
Tailwind CSS v4 via `@tailwindcss/vite`. Design tokens are CSS custom properties defined in `src/index.css` under `@theme`. Components use inline styles with `var(--color-*)` tokens rather than Tailwind utility classes directly. Dark theme only.
